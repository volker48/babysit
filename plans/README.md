## Findings considered and rejected

Recorded so future audits don't re-litigate them:

- **Exact `=` version pins in Cargo.toml flagged as unusual** — deliberate repo
  policy (matches the owner's standards); not a finding.
- **Gateway watcher token grants access to all repositories** (repo taken from
  the URL path, single global `WATCHER_TOKEN`, `gateway/src/worker.ts:141-154`)
  — real but accepted for now: single-operator deployment, wakes carry no
  payload beyond cursors. Revisit if the gateway is ever multi-tenant.
- **`dedupeExistingHistory` runs two unguarded full-table DELETE scans in every
  DO constructor** (`gateway/src/replay.ts:225-233`) — could copy the
  `intents_initialized` flag-guard pattern (`replay.ts:210-223`), but tables are
  pruned to 6h retention so scans stay small; low value.
- **Token zeroize on drop** for the Keychain bearer token — nice-to-have
  hardening; the token is short-lived in process memory and macOS-only; not
  planned.
- **`src/forge.rs` 50ms busy-poll** while waiting on child process output —
  measurable but harmless at CLI scale; not planned.
- **`prearm` writes the alarm twice per deliver/accept**
  (`setAlarm(now)` then `schedule()`, `gateway/src/replay.ts:332-334`) — extra
  storage write per webhook, by design for crash-safety (alarm is armed before
  the transaction); not a finding.
- **Automating `gateway/scripts/smoke.ts` in CI** — needs a deployed gateway
  and secrets; stays a manual runbook step.
- **Splitting `src/event.rs` (~964 lines) into client/transport modules** —
  optional tech-debt; deferred until the file needs feature work beyond
  Plan 005.
- **Direction ideas** (e.g. additional forges, richer findings output) — none
